# Dokploy CLI ‚Äî full command & flag reference

Generated by parsing `src/generated/commands.ts`

Every command also silently accepts `--json` (raw JSON output) ‚Äî omitted below since it's universal. Every generated command maps to one Dokploy API method, called as `GET` (read) or `POST` (write/action).

Format per command: `kebab-command (VERB api.method) req: --a --b ¬∑ opt: --c --d`

- `req:` = required flags (command errors without them)
- `opt:` = optional flags
- `(bool)` after a flag = boolean switch (pass `--flag` or `--flag=false`), no value otherwise takes a string
- Full invocation: `dokploy <group> <kebab-command> --flag1 val1 --flag2 val2`

**Not covered by this file** (hand-authored, not in `src/generated/commands.ts`):

- `dokploy auth -u <url> -t <token>` ‚Äî see SKILL.md

**Caveat**: flag _names_ and required/optional status are verified from source. Flag _value formats_ (e.g. whether `--domains` wants comma-separated values or repeated flags, exact enum values for something like `--applicationStatus`) are NOT verified here ‚Äî check `dokploy <group> <action> --help` or the OpenAPI spec (`repo/cli/openapi.json`) for those.
